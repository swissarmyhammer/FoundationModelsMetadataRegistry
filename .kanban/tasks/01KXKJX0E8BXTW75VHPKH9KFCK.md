## What
The second half of full Ranker adoption (selection tier is ^exv05wh). `FoundationModelsRanker` extracted this repo's retrieval pipeline — its doc comments literally cite `MetadataSearcher.retrievalSearch`/`rankEntireCatalog` and `MetadataIndex.swift`'s build step as the source — but this repo never switched over, so ~350 lines of signal computation and RRF fusion in `Sources/FoundationModelsMetadataRegistry/MetadataSearcher.swift` and the per-document precompute in `Sources/FoundationModelsMetadataRegistry/Catalog/MetadataIndex.swift` duplicate what the dependency now owns. Replace the local pipeline with Ranker's and delete the duplicates.

Ranker's API (verified in `../FoundationModelsRanker/Sources/FoundationModelsRanker/`):
- `RankedDocument(primaryText:bodyText:)` — field-for-field identical to `MetadataIndex.Entry`'s hand-rolled precompute (`weightedTermFrequency`, `termSet`, `documentLength`, primary/body trigram sets; primary = id ×`BM25.primaryFieldWeight`, body = block ×`BM25.bodyFieldWeight`).
- `HybridRanker.topMatches(ids:documents:query:cosineScores:weights:limit:) -> [Hit]` — the `retrievalSearch` shape (only signal-ranked docs, best-first, capped, ascending-index tie-break).
- `HybridRanker.fullOrdering(ids:documents:query:cosineScores:weights:) -> [Hit]` — the `rankEntireCatalog` shape (always `documents.count` hits, unranked docs follow in input order at score 0.0).
- `SignalWeights(bm25:trigram:cosine:)` — identical to the local `Weights` struct (same fields, same 1.0 defaults, same zero-weight-excludes-signal semantics).
- `CosineScoring.cosineSimilarity(_:_:)` (and vDSP `matvecScores` for batch) — identical to the local private `cosineSimilarity`.
- `Hit`/`Signals` are ALREADY the types `Match.signals` uses via the re-export, so `Hit` → `Match<Item>` mapping is id lookup only.

What stays local (deliberately — Ranker's `Searcher` does NOT replace `MetadataSearcher`): the typed `SearchableMetadata`/`Match<Item>` surface, hot reload with hash-guarded incremental re-embedding and overlapping-update merge (`incrementalBaseline`/`mergingEmbeddings` — Ranker's `Searcher` embeds only at init), and the `MetadataDiagnostic` channel. Cosine gating also stays local: `HybridRanker` skips cosine silently when `cosineScores` is nil, so the `.embeddingUnavailable` reporting and the zero-weight skip remain `MetadataSearcher`'s job.
